# Personalized AI Study Assistant

## Overview

Personalized AI Study Assistant is a Retrieval-Augmented Generation (RAG) based learning platform developed to help students study more
effectively from their own learning materials.

The system allows users to upload study documents such as PDFs, DOCX, TXT, and RTF files. Uploaded documents are processed, converted
into vector embeddings, and then stored in a FAISS vector database. Users then can ask questions related to their uploaded content and
receive context-aware answers generated by a Large Language Model (Llama 3).

In addition to question answering, the platform provides document summarization, automatic question generation, MCQ/MSQ test creation,
answer evaluation, weak-topic analysis, and personalized learning analytics.

## How It Works

### 1. Document Upload

  Users upload study material through the dashboard.

### 2. Text Extraction

  Text is extracted from uploaded documents using dedicated file loaders.

### 3. Chunking

  Large documents are split into smaller chunks for efficient processing.

### 4. Embedding Generation

  Each chunk is converted into vector embeddings using the Sentence Transformer model:

    text all-MiniLM-L6-v2 

### 5. Vector Storage

  Embeddings are stored in a FAISS index for fast similarity search.

### 6. Question Answering

  When a user asks a question:

    - The query is converted into an embedding.
    - Relevant document chunks are retrieved.
    - Retrieved context is passed to Llama 3.
    - A contextual answer is generated.

### 7. Performance Tracking

  Student interactions are analyzed to determine:

    - Accuracy
    - Weak Topics
    - Learning Level
    - Response Time

## Challenges Faced During Development

  During the development of this project, several technical challenges were encountered:

### 1. Extracting Text from Different File Formats

The system needed to support PDF, DOCX, TXT, and RTF files. Each format required a different extraction method, and some files
contained formatting issues or incomplete text extraction.

Solution: Implemented separate loaders for each file type using PyPDF, python-docx, and striprtf.

---

### 2. Handling Large Documents

Large study materials could not be directly processed by the language model due to context length limitations.

Solution: Used RecursiveCharacterTextSplitter to divide documents into overlapping chunks before generating embeddings.

---

### 3. Improving Retrieval Accuracy

Initially, retrieved document chunks were sometimes not relevant to the user's query.

Solution: Implemented semantic embeddings using Sentence Transformers and stored them in a FAISS vector index for similarity-based
retrieval.

---

### 4. Managing LLM Output Formats

The language model occasionally returned MCQs and MSQs in invalid JSON format, making parsing difficult.

Solution: Added a safe JSON parsing mechanism with validation and fallback handling.

---

### 5. Personalized Learning Analytics

Tracking student performance across different topics required storing flexible data structures.

Solution: Used MongoDB to store attempts, scores, feedback, accuracy, and topic-wise statistics.

---

### 6. Reducing Hallucinations

A standalone language model may generate answers that are not present in the study material.

Solution: Implemented Retrieval-Augmented Generation (RAG), where relevant document chunks are retrieved and provided as context before
answer generation.

---

### 7. Supporting Multiple Users

Different users needed separate document repositories and retrieval systems.

Solution: Created user-specific FAISS indexes and chunk storage directories to ensure personalized document retrieval.

---

### 8. Providing Answers When Documents Lack Information

Some user questions could not be answered from uploaded documents alone.

Solution: Integrated Tavily Search API as a fallback mechanism to retrieve relevant web information when document context is
unavailable.
